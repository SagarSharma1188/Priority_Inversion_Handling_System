Priority Inversion

Occurs when a HIGH-priority task is blocked by a LOW-priority task holding a shared mutex, while a MEDIUM-priority task runs freely — effectively inverting the intended priority order.

Priority Inheritance (PIP)

When HIGH blocks on a mutex held by LOW, LOW temporarily inherits HIGH's priority. This prevents MED from preempting LOW, so the mutex is released faster and HIGH can proceed.

Priority Ceiling (PCP)

Each mutex has a "ceiling" equal to the highest priority task that may lock it. Any task acquiring the mutex is immediately boosted to that ceiling — preventing inversion before it starts.
